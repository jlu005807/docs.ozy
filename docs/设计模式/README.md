# 🎨 设计模式 (Design Patterns)

- 📝 本目录收集设计模式的相关复习资料和速查表，主要用于开卷考试和快速回顾常用的设计模式及其示例。
- 🧩 掌握设计模式有助于编写可复用、可维护和高内聚低耦合的代码。
- 🏗️ 面向对象设计的五大基本原则（SOLID原则）是理解设计模式的基石，建议先吃透原则。
- 虽然是一门选修课，但是对于AI时代，「写代码」这件事,正在消失，未来优秀的开发者,首先是一个会做架构判断的人,其次才是会写代码的人
- 推荐学习[awesome-architecture](https://github.com/study8677/awesome-architecture)
- 同时对于设计模式的作业详见[Design_Model](https://github.com/jlu005807/Design_Model)

> [!tip]
> 不要为了使用设计模式而强行套用模式。最好的学习方式是遇到代码重构问题时，再回想一下哪种模式可以优雅地解决。

## 章节简介

- `设计模式_开卷考速查表.md`：设计模式核心知识点总结
- `design_patterns_exam_cheatsheet_examples.md`：设计模式相关示例与应用
